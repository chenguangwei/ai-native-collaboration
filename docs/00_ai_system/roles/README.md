# Skills 导航

> 两套体系并行，按团队当前阶段选择。

---

## 选哪套？

```
团队已全面拥抱 AI，不再区分前端/后端？
    → ai-native/（目标态）

团队仍按产品/研发/测试/运维分工？
    → traditional/（过渡期）
```

---

## AI-Native 体系（目标态）

> 岗位按"解决什么问题"划分，AI 处理技术细节

- **[ai-native/SKILLS_INDEX.md](ai-native/SKILLS_INDEX.md)** — 88 个技能，按 5 个 AI-Native 岗位分类
- **[ai-native/ROLE_SETUP.md](ai-native/ROLE_SETUP.md)** — 岗位配置 + 决策树

4 个岗位：`delivery-engineer` · `ai-engineer` · `quality-engineer` · `product-owner`

---

## 传统体系（过渡期）

> 保留现有分工，人人配备 AI 工具，映射到 AI-Native 角色值

- **[traditional/SKILLS_INDEX.md](traditional/SKILLS_INDEX.md)** — 技能按传统岗位分类
- **[traditional/ROLE_SETUP.md](traditional/ROLE_SETUP.md)** — 传统岗位 → currentRole 映射

3 个角色值：`product-owner` · `delivery-engineer` · `quality-engineer`


---

## 通用技能（两套体系共享）

所有岗位都适用的核心工作流：

| 场景 | 命令 |
|------|------|
| 需求不清楚 | `/deep-interview` |
| 开始新功能 | `/brainstorming` |
| 遇到 Bug | `/systematic-debugging` |
| 完成前验证 | `/verification-before-completion` |
| 代码审查 | `/review` |
| 发布检查 | `/ship` |
| 更新 OMC | `/omc-upgrade` |
