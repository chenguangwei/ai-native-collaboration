## 常用命令速查

### 创意工作
```
/brainstorming  → 开始任何创意工作前必须
/plan-ceo-review  → 审视产品方向
/design-consultation → 设计咨询
```

### 开发工作
```
/test-driven-development → 开始实现前
/writing-plans → 制定计划
/executing-plans → 执行计划
/verification-before-completion → 声称完成前
```

### 代码质量
```
/review → 提交前审查
/ship → 发布前检查
/audit → 全面审计
```

### 调试问题
```
/systematic-debugging → 遇到 bug
/investigate → 根因调查
/guard → 启用安全模式
```

| 场景 | 触发技能 |
|------|----------|
| 创建新功能 | brainstorming → plan-* → TDD → verification |
| 修复 bug | systematic-debugging → verification |
| 代码审查 | review → receiving-code-review |
| 发布 | ship → qa → verification → ship |
| 设计迭代 | design-review → 迭代技能 (adapt, polish, etc.) |

## IBelick UI Skills

> 来源: [ibelick/ui-skills](https://github.com/ibelick/ui-skills)

### 使用方法

```bash
# 审查 src/ 目录
/baseline-ui review src/

# 修复无障碍问题
/fixing-accessibility audit src/

# 修复元数据
/fixing-metadata audit src/

# 审查动画性能
/fixing-motion-performance audit src/
```

### 可用技能

| 技能 | 命令 | 用途 |
|------|------|------|
| Baseline UI | `/baseline-ui` | 固执己见的 UI 基线检查 |
| Fixing Accessibility | `/fixing-accessibility` | 键盘、标签、焦点、语义 |
| Fixing Metadata | `/fixing-metadata` | 标题、meta、社交卡片 |
| Fixing Motion Performance | `/fixing-motion-performance` | 安全、性能优先的动画 |
