# /deep-interview

在执行复杂任务前，用 Socratic 提问法将模糊想法转化为清晰规格。

## 使用场景

**必须使用**：
- 用户需求模糊，没有具体文件路径、函数名、验收标准
- 用户说"帮我做一个..."、"我想要..."但没有细节
- 任务超过 3 个实现步骤且需求不明确

**不需要使用**：
- 用户有具体的文件路径、函数名、明确的验收标准
- 用户说"直接做"、"跳过问题"
- 简单的 Bug 修复或单一改动

## 执行流程

### Phase 1: 初始化

1. 分析用户输入，判断是**新建项目（greenfield）**还是**改造已有代码（brownfield）**
2. 如果是 brownfield：读取相关代码文件，获取上下文（不要问用户他在代码里就有答案的问题）
3. 初始化清晰度评分：所有维度从 0 开始

公告给用户：
```
开始深度需求访谈。我会一次问一个问题，每次回答后展示清晰度评分。
当歧义度降到 20% 以下时，我们开始执行。

你的需求："{用户输入}"
当前歧义度：100%（刚开始）
```

### Phase 2: 访谈循环

每轮执行：

**2a. 生成问题**
- 找到当前评分最低的维度
- 针对该维度生成一个精准问题
- 每次只问一个问题

**维度和问题风格**：
| 维度 | 问题风格 | 示例 |
|------|---------|------|
| 目标清晰度(40%) | "具体会发生什么？" | "说'管理任务'，用户的第一个操作是什么？" |
| 约束清晰度(30%) | "边界在哪里？" | "需要离线支持，还是必须联网？" |
| 验收标准(30%) | "怎么知道做完了？" | "如果我展示给你，你会说什么让你觉得'对了'？" |

**2b. 使用 AskUserQuestion 展示问题**：
```
第 {n} 轮 | 聚焦维度：{最弱维度} | 当前歧义度：{分数}%

{问题}
```

**2c. 评分**（收到回答后）：

为每个维度打分 0.0-1.0：
- **目标清晰度**：核心目标能否用一句话表达？
- **约束清晰度**：边界和非目标是否明确？
- **验收标准**：能否写出可执行的测试？

计算：`歧义度 = 1 - (目标×0.40 + 约束×0.30 + 标准×0.30)`

**2d. 展示进度**：
```
第 {n} 轮完成

| 维度 | 得分 | 权重 | 加权得分 | 仍不明确 |
|------|------|------|---------|---------|
| 目标 | {s} | 40% | {s*0.4} | {gap} |
| 约束 | {s} | 30% | {s*0.3} | {gap} |
| 标准 | {s} | 30% | {s*0.3} | {gap} |
| **歧义度** | | | **{score}%** | |

{score ≤ 20% ? "清晰度达标！准备执行。" : "下一问聚焦：{最弱维度}"}
```

**2e. 退出条件**：
- 歧义度 ≤ 20%：自动进入 Phase 4
- 轮数 ≥ 3 且用户说"够了"/"直接做"：警告后进入 Phase 4
- 轮数 ≥ 10：强制进入 Phase 4，标注剩余歧义

### Phase 3: 挑战模式（特殊轮次激活）

- **第 4 轮+：反向模式** — 挑战核心假设："如果不需要这个约束呢？"
- **第 6 轮+：简化模式** — 寻找最小可行版本："最简单但仍有价值的版本是什么？"

每种模式只使用一次。

### Phase 4: 生成规格文档

将完整访谈结果写入：`docs/00_ai_system/specs/{slug}-spec.md`

**规格文档结构**：
```markdown
# 需求规格：{标题}

## 元数据
- 访谈轮数：{n}
- 最终歧义度：{score}%
- 类型：greenfield/brownfield
- 生成时间：{timestamp}

## 清晰度明细
| 维度 | 得分 | 权重 | 加权 |
|------|------|------|------|
| 目标清晰度 | | 40% | |
| 约束清晰度 | | 30% | |
| 验收标准 | | 30% | |

## 目标
{一句话描述核心目标}

## 约束
- {约束 1}
- {约束 2}

## 非目标（明确排除）
- {排除项 1}

## 验收标准
- [ ] {可测试的标准 1}
- [ ] {可测试的标准 2}

## 暴露的假设
| 假设 | 如何挑战 | 决策 |
|------|---------|------|
| | | |

## 访谈记录
<details>
<summary>完整问答（{n} 轮）</summary>
### 第 1 轮
**问：** ...
**答：** ...
**歧义度：** X%
</details>
```

### Phase 5: 执行桥接

规格写完后，用 AskUserQuestion 询问：
- 选项 1：直接执行（我来实现规格里的内容）
- 选项 2：先给我看规格，我来确认
- 选项 3：继续访谈（当前歧义度 X%，还可以更清晰）

执行前确认规格文件已保存。
