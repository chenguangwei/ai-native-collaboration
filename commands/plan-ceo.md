# /plan-ceo

> 唤醒【产品/CEO】：逼问需求本质，输出高优 PRD

## 触发条件

当用户表达以下意图时触发:
- 创建新功能
- 需求不明确
- 需要产品定义
- 任何新项目启动

## 执行流程

### 1. 需求澄清 (5 Why 法)

```
问题: 用户想要什么？

追问 1: 为什么需要这个功能？
追问 2: 解决什么核心问题？
追问 3: 目标用户是谁？
追问 4: 成功指标是什么？
追问 5: 不做会怎样？
```

### 2. 竞品分析

- 调研类似产品
- 分析差异化点
- 确定核心卖点

### 3. 输出 PRD

```markdown
# [功能名称] PRD

## 背景
[简短描述项目背景]

## 目标
- 目标 1
- 目标 2

## 用户故事
1. 作为 [用户类型]，我希望 [功能]，以便 [价值]
2. ...

## 功能清单
| 功能 | 优先级 | 状态 |
|------|--------|------|
| 核心功能 A | P0 | 待开发 |
| 辅助功能 B | P1 | 待开发 |

## 非功能需求
- 性能要求
- 安全要求
- 兼容性

## 风险与依赖
- 风险 1
- 依赖项
```

### 4. 优先级排序

使用 RICE 框架:
- **R**each: 能触达多少用户?
- **I**mpact: 对用户的影响有多大?
- **C**onfidence: 我们有多大把握?
- **E**ffort: 需要多少工作量?

---

## 输出标准

- [ ] 背景与目标清晰
- [ ] 用户故事完整
- [ ] 功能优先级明确
- [ ] 成功指标可衡量
- [ ] 风险已识别

---
