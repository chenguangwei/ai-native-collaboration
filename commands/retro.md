# /retro

> 唤醒【敏捷教练】：每周复盘踩坑记录，自动提取为新 Skill

## 触发条件

- 每周固定复盘
- 项目里程碑完成
- 重要问题解决后

## 执行流程

### 1. 收集数据

#### 1.1 读取近期记忆

- `/memory/today.md` - 每日工作
- `/memory/active-tasks.json` - 任务状态

#### 1.2 收集问题

```
本周遇到的问题:
1. 数据库连接池耗尽
2. 前端构建失败
3. API 响应超时
```

### 2. 分类分析

#### 2.1 问题分类

| 类别 | 数量 | 示例 |
|------|------|------|
| 技术 | 3 | 数据库性能 |
| 流程 | 1 | 代码审查延迟 |
| 沟通 | 1 | 需求变更 |

#### 2.2 根因分析

```
问题: 数据库连接池耗尽
根因: 未正确释放连接
教训: 每次查询后必须显式关闭连接
```

### 3. 提取经验

#### 3.1 成功经验

- 自动化测试节省时间
- Code Review 拦截问题

#### 3.2 失败经验

- 过度设计导致返工
- 缺少文档影响协作

### 4. 生成报告

```markdown
# 📅 2024-W01 复盘报告

## 概览
- 总工作日: 5
- 完成任务: 12
- 新发现问题: 5

## 👍 做得好的
1. TDD 流程执行到位
2. 代码审查及时
3. 文档更新及时

## 👎 需要改进
1. 技术选型过于复杂
2. 估算时间不准确
3. 缺乏性能监控

## 🔧 改进措施
- [ ] 简化技术栈
- [ ] 添加性能监控
- [ ] 改进估算方法

## 📚 学到的经验
1. 永远不要假设 API 稳定
2. 单元测试是最佳文档
3. 早提交小改动
```
