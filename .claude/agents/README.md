# 🎭 Agents 智能助手目录

> **注意：此目录现已精简为 OMC 原生 Agent 的专属软链接存储区。**

随着 AI-Native Scaffold 的演进，我们彻底废弃了原有的零散中文独立 Agent，以避免与强大的全局 `skills` 系统发生职责冲突。

## 核心架构原则

1. **统一技能库**: 所有的通用审查标准、防坑规则（如 React、Java、Python 的特定架构模式、性能优化手册等）均已直接下沉至 `.agents/skills/` 之中。
2. **单一中枢**: 代码检查、自动修复、计划生成等动作，请统一使用 `omc` 提供的标准命令（例如 `/omc-plan`、`/review`、`/qa`）或其他顶级宏命令。
3. **自动调度**: OMC 系统在运行任务时，不仅会调度本目录下的官方代理（如 `omc-code-reviewer.md` 等），还会**自动读取 `skills` 下的专属能力**进行合并判断。

## 目录现状

本目录下的所有 `.md` 文件（如 `omc-*.md`）均为指向 `.agents/skills/omc/agents/` 或其内部资源目录的软链接。

**请勿在此直接新增松散的 `.md` Agent 提示词文件。** 若有业务检查规则需要补充，请前往 `.agents/skills` 目录编写新的 Skill 并在其中定义完善的验证工作流。
