# 🔄 AI 上下文记忆与交接机制 (Memory Sync)

> 这是 AI 工作流中极其重要的一环。在多轮对话或跨角色协作时，大模型的记忆窗口可能会发生断层。为了保证高连贯性，**这套交接机制是强制执行的**。

## 📍 绝对触发条件 (Must Trigger)

在以下任何一种情况发生时，你**必须**暂缓手中的代码操作，优先将当前进度同步到 `memory/`：
1. **完成了一个重大阶段点（Milestone）** 准备向用户汇报。
2. **遇到了严重的阻碍（Blocker）**，短期内无法解决，准备让用户介入。
3. **完成了一个会话（Session）所有的目标任务**，准备结束工作或准备并入主干。
4. 你即将切换上下文，去做另一件不相关的事情。

## 💾 三重记忆表 (The 3-Tier Memory)

位于 `项目根目录/memory/`，全以 Markdown 形式扁平组织，供 AI 自主读写：

### 1. 📋 `active-task.md` (主线阶段待办)
这是宏观的清单。当接手一个新的复杂 Feature 需求时，**如果还没有建立拆站任务树，优先来这里追加 Checklist。**
- 完成了其中一项就打个 `[x]`。
- 绝不在这记录长篇大论，只记录待办树状图。

### 2. 🤝 `handoff.md` (会话交接单)
这是微观的工作流日志，是新对话接力必须要读的文件！每次执行上述“触发条件”时，执行对它的追加或更新写入：
- **落款署名（强制必须）**：必须以你的当前身份角色（或者你正在处理的任务领域）和当前系统日期时间作为每一条留言的开头前缀。例如：`**[前端工程师 - 2026-03-28 11:30]**`。
- **刚才干了什么**：极简地罗列“修复了 A模块的某 Bug”、“写了 B 模块的测试”。
- **留了什么坑 / Blocker**：“因为 C 库的版本冲突，目前暂时通过 mock 绕过”。
- **下一步交给谁干**：“明天请接手实现 Auth 接口，[等待后端联调]”。
*(注意：请保持它的大小在 100 行以内，旧的、已完结的流水账直接清空即可，不需要像履历一样堆叠，一切代码变动看 Git !)* 

### 3. 🧠 `project-facts.md` (踩坑与准则备忘)
当你深刻认识到这个项目的一些隐秘规则时（比如“本系统里的用户 ID 必须使用 UUID v4 强校验”，“千万不要动 Dockerfile 里的 Node 版本”），把它以**一条原则**的形式添加到这里。
这是一本项目的“暗黑生存法则”。

---

**核心原则**：你是自治的 AI 工程师。你写 `memory/` 是写给下一个“你”看的，用**最高度概括的技术语言**，不要有废话。不要去维护任何繁重的 JSON 锁，用你的双眼读去 Markdown。
